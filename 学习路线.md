# 本科就业学习路线（211 大二 · 从零到 Offer）

> 适用对象：211 计算机/软件工程本科，大二在读，目标本科就业  
> 当前基础：已用 AI 完成一个 Next.js 全栈项目  
> 预计周期：大二下 → 大四秋招，约 2 年

---

## 你的优势与劣势

**优势：**
- 211 学历，简历不会被大厂直接筛掉
- 大二就有全栈项目 + GitHub，起点高于大多数人
- 已接触 AI 辅助编程，效率远超传统学习方式

**劣势：**
- 底层基础薄弱（能用但讲不清原理）
- 没有算法训练
- 时间紧——大三上就要开始投实习

---

## 第一阶段：补地基（大二下，2-3 个月）

### 1.1 JavaScript 核心（4 周）

**免费资源：**[现代 JavaScript 教程](https://zh.javascript.info/)（全局最佳中文 JS 教程）

| 周次 | 内容 | 输出 |
|------|------|------|
| 第 1 周 | 基础：变量、类型、函数、对象 | 能写纯 JS 计算器 |
| 第 2 周 | 核心：原型链、this、闭包、作用域 | 能画图解释原型链 |
| 第 3 周 | 异步：Promise、async/await、Event Loop | 能画图解释 Event Loop |
| 第 4 周 | 模块化、错误处理 | 用 Node.js 写一个命令行工具 |

**面试常考：** 闭包是什么？`Promise.all` 怎么实现？Event Loop 执行顺序题。

### 1.2 数据结构（4 周，每天 2 小时）

**免费资源：**[Hello 算法](https://www.hello-algo.com/)（中文、图解、代码可运行）

| 周次 | 内容 | 配套 LeetCode |
|------|------|--------------|
| 第 1 周 | 数组、链表、栈、队列 | 206, 20, 232 |
| 第 2 周 | 哈希表、二叉树、堆 | 1, 94, 104 |
| 第 3 周 | 图、排序算法 | 200, 215 |
| 第 4 周 | 递归、回溯、动态规划入门 | 70, 46, 78 |

**关键：** 不要只看书，每学一个数据结构就刷 3 道题。

### 1.3 TypeScript 系统学（1 周）

你已经写过 TS 了，这次系统补：
- 泛型 `generics`（面试高频）
- 工具类型：`Partial<T>`、`Pick<T>`、`Omit<T>`、`Record<K,V>`
- 类型守卫、条件类型

**资源：** [TypeScript 官方文档](https://www.typescriptlang.org/docs/handbook/)，看 Handbook 部分就够了。

### 1.4 计算机网络（2 周）

只看面试会考的：

| 知识点 | 程度 |
|--------|------|
| HTTP/HTTPS | 三次握手、四次挥手、状态码、缓存控制 |
| DNS | 域名解析过程 |
| 跨域 | CORS 原理、如何解决 |
| Cookie/Session/Token | 三者的区别和使用场景 |

**资源：** 《图解 HTTP》+ 小林 coding 图解网站

## 第二阶段：搞懂你项目里的东西（大二下，1 个月）

不是写新项目，是把 Mini Mall 每一行代码吃透。

### 2.1 认证系统（第 1 周）

**打开 `src/lib/auth.ts`，搞懂：**
- JWT 由 Header + Payload + Signature 三部分组成
- `jose` 库怎么签发和验证 token
- 为什么用 httpOnly Cookie 而不是 localStorage
- bcrypt 为什么安全（盐值 + 迭代次数）

**面试题：** 如果有人偷了你的 JWT，他能伪装成你吗？怎么防？

### 2.2 数据库 + Prisma（第 2 周）

**打开 `prisma/schema.prisma`，搞懂：**
- 一对多、多对多关系怎么设计
- `onDelete: Cascade` 是什么意思
- 为什么价格用 Decimal 而不是 Float

**打开 `src/app/api/orders/route.ts`，搞懂：**
- 事务的 ACID（原子性、一致性、隔离性、持久性）
- 为什么下单必须用 `$transaction`
- 库存扣减的并发问题

### 2.3 React + Next.js（第 3 周）

**搞懂：**
- Server Component vs Client Component
- `useState` 和 `useEffect` 的原理
- App Router 的 layout、page、route 怎么协同
- 为什么 `middleware.ts` 能做路由守卫

### 2.4 练手：关掉 AI，给 Mini Mall 加一个功能（第 4 周）

比如加一个"商品收藏"功能。从数据库设计 → API → 页面，全程不看我之前写的代码。卡住了先想 10 分钟，再查文档，最后求助。

## 第三阶段：LeetCode 专项突破（大二暑假，8 周）

这是就业最关键的一环。211 学历 + 200 道题 = 大厂面试入场券。

### 刷题计划

**资源推荐：** [代码随想录](https://programmercarl.com/)（比 LeetCode 官方按顺序刷高效 10 倍）

| 周次 | 专题 | 题目数 |
|------|------|--------|
| 第 1-2 周 | 数组、链表 | 30 题 |
| 第 3 周 | 哈希表、字符串 | 25 题 |
| 第 4 周 | 栈与队列、二叉树 | 30 题 |
| 第 5 周 | 回溯算法 | 20 题 |
| 第 6 周 | 贪心、动态规划入门 | 25 题 |
| 第 7 周 | Hot 100 集中刷 | 30 题 |
| 第 8 周 | 复习 + 限时模拟 | 回顾错题 |

### 刷题铁律

1. **一道题不超过 30 分钟**，想不出来直接看答案，看懂后自己写一遍
2. **每道题写解题注释**，面试时要能讲清楚思路
3. **定期回顾**，每隔两周回头看之前的题
4. **不要追求题量，追求"看到题就知道用什么解"**

## 第四阶段：实习准备（大三上，9-11 月）

### 4.1 做第 2 个项目（4 周）

不能再用 AI 生成了。自己设计、自己写。推荐做一个有"用户"的项目：

**推荐方向：** 校园论坛 / 二手交易平台 / 课程评价系统

要求：
- 自己从零搭，不复制 Mini Mall
- 有注册登录、发帖/评论、点赞、搜索
- 部署到线上（Vercel 免费）
- README 写清楚、commit 历史干净

### 4.2 写简历（1 周）

简历只说三件事：**你做了什么、用了什么技术、解决了什么问题**

示例（Mini Mall）：
> **Mini Mall 微型电商** - Next.js + TypeScript + Prisma  
> • 独立完成全栈电商项目，包含商品浏览、购物车、订单管理等 10+ 页面  
> • 实现 JWT 认证 + 中间件路由守卫，以及基于事务的下单流程  
> • 设计心悦会员三级折扣系统，支付后自动累加消费并升级等级

### 4.3 投日常实习

| 渠道 | 说明 |
|------|------|
| **Boss 直聘** | 体量最大，中小厂最集中 |
| **牛客网** | 大厂内推、面经、笔试真题 |
| **企业官网** | 直接投，字节/腾讯/阿里都有校招入口 |
| **学长学姐内推** | 211 的优势，校友网络比什么都好使 |

**关键：** 日常实习门槛远低于暑期实习和秋招。先上车，有了一段实习经历，后续会容易非常多。

## 第六阶段：秋招冲刺（大四上，7-10 月）

### 秋招时间线

```
7 月    提前批开始（字节、腾讯最早）
8 月    正式批大量开放
9 月    笔试 + 面试高峰期
10 月   补录 + 谈 offer
```

### 面试全流程

```
简历筛选 → 笔试（算法 3-5 题）→ 一面（技术）→ 二面（技术+项目）→ HR 面 → Offer
                                                                   ↓
                                                              三面（部分大厂）
```

### 面的什么

| 轮次 | 考察内容 | 占比 |
|------|---------|------|
| 一面 | 算法（1-2 题）+ 基础知识 | 算法 40% 基础 60% |
| 二面 | 项目深挖 + 场景设计 | 项目 70% 算法 30% |
| HR 面 | 沟通能力、职业规划 | 一般不挂人 |

| 时间 | 检查项 | 没做到怎么办 |
|------|--------|-------------|

| 大四秋招 | 拿至少 1 个 Offer | 没拿到就继续投春招，千万别放弃 |

## 最重要的三句话

1. **实习是本科就业最大的加分项——比 GPA、比赛、证书都好使。** 大三无论如何要找一段实习。

2. **LeetCode 不是刷越多越好，是"看到题目就知道思路"才好。** 200 道精刷效果远超 500 道囫囵吞枣。

3. **你已经走在前面了。** 大二就有全栈项目、有 GitHub、会用 AI 工具，这些都是真实竞争力。保持节奏，别慌。
