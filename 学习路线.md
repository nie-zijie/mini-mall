> 核心策略：**不追求学完，追求能面试**

> 预计周期：大二 6 月 → 大三 9 月投实习，约 3.5 个月

---

```
现在：6 月中旬
大三上开学：9 月

暑假优势：每天可以学 6-8 小时（在校期间 2-3 小时）

```

## 第一阶段：JS 速通（第 1-4 周，6 月到 7 月中旬）

> 目标：JavaScript 基础语法 + 核心概念，够看懂你的项目代码

### 第 1-2 周：基础语法（每天 3 小时）

**资源：**[现代 JavaScript 教程](https://zh.javascript.info/) 第一章到第五章

用 VS Code 建 `test.js`，边看边敲。不要看视频，文档效率高 3 倍。

```
day 1-2   → 变量、类型、运算符
day 3-4   → 条件判断、循环
day 5-6   → 函数、箭头函数、作用域
day 7-8   → 对象、数组、解构
day 9-10  → map / filter / reduce / sort
day 11-12 → Promise、async/await、fetch
day 13-14 → 回头复习，把前两天模糊的再过一遍
```

### 第 3 周：核心概念（这三个面试必问）

| 知识点 | 目标 | 动手 |
|--------|------|------|
| 闭包 | 能写一个计数器闭包，能画图解释 | 写 3 个不同场景的闭包例子 |
| 原型链 | 知道 `prototype` 和 `__proto__` | 画一张原型链图 |
| Event Loop | 能解释为什么 `setTimeout 0` 不是立即执行 | 写 5 个异步顺序题，预测输出 |

**补充资源：** B 站搜"技术蛋老师 闭包"、"技术蛋老师 Event Loop"，每个视频 15 分钟，画动画讲的，一看就懂。

### 第 4 周：用你的项目练手

**打开 Mini Mall 的 `src/app/(shop)/cart/page.tsx`，试着：**
- 找到 `useState` 是哪行，它在干什么
- 找到 `useEffect` 是哪行，它什么时候执行
- 找到 `fetch("/api/cart")` 是哪行，数据从哪来

看不懂没关系，标注出来问我。这一周的目标不是"全看懂"，是"知道哪里看不懂"。

## 第二阶段：React + TypeScript 速通（第 5-7 周，7 月中旬到 8 月初）

> 目标：看懂你的 Mini Mall 里的 React 代码

### 第 5 周：React 核心

**资源：**[React 官方快速入门](https://react.dev/learn)（只看"Describing the UI"和"Adding Interactivity"两章）

**动手：** 做一个计数器 + Todo List。

### 第 6 周：TypeScript 基础

你已经写过 TS 了。这次补类型系统。

**资源：**[TypeScript Handbook](https://www.typescriptlang.org/docs/handbook/) 基础类型 + 接口 + 泛型入门

**动手：** 把你 Mini Mall 里的 `src/types/index.ts` 打开，确认每个类型定义你都能看懂。

### 第 7 周：Next.js 核心概念

**打开你的 `src/app/`，搞懂三个问题：**

1. 为什么有的文件有 `"use client"`，有的没有？
2. `layout.tsx` 和 `page.tsx` 有什么区别？
3. `src/app/api/` 下面的 `route.ts` 是怎么被访问到的？

**资源：**[Next.js Learn](https://nextjs.org/learn) 的 Dashboard 教程（2 天看完）

## 第三阶段：吃透你的项目（第 8-10 周，8 月）

> 目标：把 Mini Mall 变成你自己的——能一行行讲出每个文件在做什么  
> 这是你面试的核心武器

### 第 8 周：认证系统吃透

```
文件清单：
  src/lib/auth.ts         → JWT 签发、验证、Cookie 管理
  src/middleware.ts        → 路由守卫
  src/app/api/auth/login/route.ts     → 登录逻辑
  src/app/api/auth/register/route.ts  → 注册逻辑
```

**你要能回答：**
- JWT 由哪三部分组成？
- 为什么用 httpOnly Cookie 存 token？
- 如果有人偷了你的 Cookie，他能登录吗？
- bcrypt 为什么安全？

### 第 9 周：数据库 + 接口吃透

```
文件清单：
  prisma/schema.prisma    → 6 个表的关系
  src/app/api/orders/route.ts  → 下单事务逻辑
  src/app/api/products/route.ts → 商品搜索+分页
```

**你要能回答：**
- User 和 Order 是什么关系？（一对多）
- 下单为什么用 `$transaction`？
- `onDelete: Cascade` 是什么意思？
- 为什么价格用 Decimal 而不是 Float？

### 第 10 周：会员系统 + 反应试准备

```
文件清单：
  src/lib/member.ts       → 等级计算逻辑
  src/app/api/orders/[id]/route.ts → 支付 + 等级升级
```

**你要能回答：**
- 用户支付后，谁能保证他的 totalSpent 一定增加？
- 事务里先更新订单还是先更新用户？顺序重要吗？
- 如果你的项目有一万个并发用户同时下单，哪里会出问题？

## 第四阶段：算法入门 + 简历（第 11-13 周，8 月底到 9 月中旬）

> 目标：LeetCode 50 题 + 简历能投

### 算法（每天 1-2 小时）

**只刷这些够了（日常实习不考难的）：**

```
数组：Two Sum、三数之和、最大子数组和
链表：反转链表、合并链表、环形链表
哈希表：两数之和、字母异位词
栈：有效括号、最小栈
二叉树：前中后序遍历、最大深度
动态规划：爬楼梯、打家劫舍（入门 2 题）
```

**50 题，按这个顺序刷。** 跟 [代码随想录](https://programmercarl.com/) 走。

**刷题铁律：**
- 15 分钟想不出 → 看答案 → 关掉答案自己写一遍
- 每道题写一行注释说你用了什么方法
- 同一道题隔三天再做一遍

### 简历（第 13 周）

你的简历核心是 Mini Mall 这一个项目。只要讲透，一个项目就够。

**项目描述示例：**

> **Mini Mall 微型商城** — Next.js 16 + TypeScript + Prisma + SQLite
> - 独立完成全栈电商平台，含商品浏览搜索、购物车、订单管理、后台管理等 20+ 页面
> - 实现 JWT 认证 + 中间件路由守卫，Prisma 事务保证下单的库存安全和数据一致性
> - 设计心悦会员三级折扣系统，支付后自动累加消费金额并动态升级等级

**简历三原则：**
1. 一页纸，不要两页
2. 每个项目三行字：是什么 + 用了什么 + 有什么亮点
3. 不会讲的别往上写——面试官会顺着简历问

### 面试会问什么

```
一面（技术面，45 分钟）：
  - 自我介绍（1 分钟）
  - 算法题 1-2 道（Easy 到 Medium）
  - JS 基础：闭包、Promise、Event Loop
  - React 基础：useState、useEffect、虚拟 DOM
  - 你的项目：让你介绍 Mini Mall，追问技术细节

二面（如果面大厂才有）：
  - 再加一道算法题
  - 项目深挖：为什么用 Prisma？为什么用 Next.js 而不是 Vite？
  - 场景题：如果你要给你的项目加一个秒杀功能，你打算怎么做？

HR 面：
  - 什么时候能到岗？能实习多久？一周几天？
```

### 面对第一个面试

你第一次面试大概率会挂。**挂了≠你不行，=你需要补面试经验。**

每场面试完立刻记下来：
- 哪个题没答上来？
- 面试官追问了项目的什么细节？
- 下次被问到同一个问题怎么回答？
